# Contributing to Clear

Thank you for your interest in contributing to Clear. The language is in early design (v0.1), which means your input has outsized impact on the final shape of the language.

## What We Need

| Area | How to Help |
|------|-------------|
| Language design | Propose new keywords, syntax changes, or semantic rules via Issues or Discussions |
| Examples | Write `.clear` files that demonstrate real-world use cases |
| Specification | Help formalize grammar rules in `spec/` |
| Documentation | Improve explanations, fix typos, add diagrams |
| Tooling ideas | Propose CLI commands, editor integrations, or developer workflows |

## How to Contribute

1. **Check existing issues and discussions** to avoid duplicates.
2. **Open a Discussion** for broad proposals or questions.
3. **Open an Issue** for specific bugs or well-defined feature requests.
4. **Submit a Pull Request** for concrete changes (examples, spec edits, docs).

## Pull Request Guidelines

- Keep PRs focused. One proposal per PR.
- For spec changes, explain the rationale in the PR description.
- For examples, ensure the `.clear` file is self-explanatory and follows existing conventions.
- Use clear commit messages: `Propose: ...`, `Fix: ...`, `Add: ...`, `Docs: ...`.

## Code of Conduct

This project follows the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). Be respectful, constructive, and inclusive.

## Questions?

Open a [Discussion](https://github.com/sahin/clear/discussions) — there are no bad questions at this stage.
